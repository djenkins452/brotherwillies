Rollback Procedure

One-step rollback: set USE_DYNAMIC_RATINGS=false in Railway env vars. Effective immediately on next request — no code change, no DB migration.
For a permanent rollback (re-default the repo): revert the commit that flipped the default in brotherwillies/settings.py and push.
After rollback, capture a snapshot with notes "post-rollback" so the historical ledger preserves the timeline.

Diagnostic only. This page does not modify any recommendation, threshold, or calibration constant. Capturing snapshots (via the management command) is the only state-modifying operator action available from this surface.
