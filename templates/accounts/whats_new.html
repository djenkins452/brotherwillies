Apr 19, 2026 — Guided Choice & Odds Integrity

Evolve from "here are the games" to "here's the single best spot on the board — and why — backed by real data."

Primary vs Secondary actions

Action tags now carry a strength. When both 💰 Best Bet and 🔥 Watch Now fire on the same game, Best Bet is the Primary action (filled pill) and Watch Now becomes Secondary (outlined). Exactly one Primary per game. Nothing is hidden — the hierarchy just makes the best spot obvious.

★ Top Opportunity (scarcity by design)

Across every live and today's tile, only the single highest-confidence Best Bet gets a gold ★ Top Opportunity tag. Scarcity is the point. One tag you can trust beats ten you have to second-guess. Configurable via settings.MLB_MAX_TOP_OPPORTUNITIES if you want more.

Line-value signal (real edge)

New signal: line_value_discrepancy = |house_prob − market_prob|. When the gap is ≥ 6%, the game gets a Model edge vs. market reason and Best Bet prefers this over a tight-spread trigger. This is a real "we see something the market doesn't" signal, not just a thin-line bet.

Structured reasons (no more UI strings in the service layer)

Reasons are now structured keys (tight_spread, ace_matchup, line_value, late_game…) and the UI maps them to labels at render time. Same key can render differently in the tile vs. help modal vs. analytics without any service-layer change.

Fail-fast odds pipeline

We never silently operate with zero odds any more. The API client rejects HTML responses served as 200s (rate-limit pages, WAF challenges) with an explicit error that includes URL, status, and a body snippet. ingest_odds raises in DEBUG when zero snapshots are created and logs a high-severity error in prod. A post-ingest sanity check catches the case where rows wrote but no games-of-today got coverage.

Apr 19, 2026 — MLB Tile Context, Bet Dropdown & Pending Indicator

Denser tiles, smarter mock-bet flow, and a clear visual signal for bets already in play.

Everything that matters, on one tile

MLB tiles now show each team's season record (e.g. 12-8), a colored streak chip for runs of 2+ games ("W3" green / "L2" red), and each starting pitcher's W-L record next to their name. Live tiles gained the pitcher matchup row too. The numbers come from existing data — no new API calls — and the recent-form streaks are computed from final games in a single batched query per page render.

Selection is a dropdown, not a text field

Mock Bet's Selection is now a two-option dropdown driven by the game context. Moneyline lists both teams. Spread lists both sides of the run line with signs attached. Total lists Over/Under with the snapshot's total. Changing the bet type re-populates the dropdown. Type-to-edit still works for sports that don't pass selection options.

🎯 Pending bet indicator

If you have a pending mock bet on a game, its tile carries a target icon in the header and a thin green rail down the right edge. One batched query on hub render, skipped entirely for anonymous viewers.

Apr 19, 2026 — MLB Action Layer

Turn insight into action — action tags tell you what to do, and one click places a mock bet with everything pre-filled.

🔥 Watch Now   💰 Best Bet

Tiles on the MLB Hub now carry up to two action tags. Watch Now lights up for live games that are close or for ace pitching matchups that haven't turned into blowouts. Best Bet lights up when the market spread is tight and both starters are known — we deliberately suppress it when a pitcher is TBD, because the model has less to work with.

One-click mock bet

Every tile (for logged-in users) has a Mock Bet button. Clicking it opens the bet modal with the selection, bet type, and odds already populated: the team with the higher-rated starter is pre-selected, the bet type flips to spread when the market is within ±1.5, and moneyline odds carry over from the latest snapshot. You can still override anything before submitting.

Architecture

All decisions (which action to show, which team to default to) live in a pure service function — not the template. Templates render what they're handed. No new endpoints; the existing mock-bet AJAX flow is reused.

Apr 19, 2026 — MLB Hub: Tile Priority Layer

A priority-driven command center for real MLB games — built to answer "what should I care about right now?" at a glance.

Tiles for what matters now, list for later

The MLB Hub now shows Live and Today's games as horizontally-scrolling tiles at the top, sorted by a new priority score. Games beyond today stay in a chronological list below — so the contrast between "now" and "later" is visual, not buried in a scroll.

Priority chips tell you why

Tiles can surface an amber Priority chip or a grey Watch chip. A short "why" line on each tile calls out the trigger: close spread, tight live score, key injury, ace pitching matchup. Blowout live games and TBD-pitcher games get demoted automatically.

Designed to extend

The signals layer lives in a dedicated service with a single weight table. User favorites, odds movement, and playoff-importance signals are already wired as seams — they'll switch on as the underlying data lands without a page-wide rewrite. Inning-based live sort is queued for when the ingestion pipeline captures inning state.

Apr 19, 2026 — Baseball Expansion: MLB + College Baseball

Live MLB and D1 college baseball data, a pitcher-weighted prediction model, and mock bets on real games.

MLB is now live

Visit the MLB Hub or the Lobby MLB tab to see today's games, probable pitchers, and live scores. Data is pulled directly from the official statsapi.mlb.com endpoint — no API key required, no simulated data, no filler. If a pitcher hasn't been announced yet, the card says "Probable pitcher TBD" rather than making something up.

College baseball (full D1)

The College Baseball Hub and Lobby tab cover all NCAA Division I programs via ESPN's public scoreboard. Coverage is schedule + live scores + finals; ESPN doesn't expose probable pitchers for college baseball, so those are marked TBD and the model's confidence reflects it.

Pitcher-weighted prediction model

The baseball house model weights the starting-pitcher matchup at 65%, team rating at 35%, plus a modest home-field adjustment (2.5 for MLB, 2.0 for college baseball). Pitcher ratings are derived from season ERA, WHIP, and K/9 on a 10–95 scale. If either starter is unknown, pitcher contribution is zero and confidence drops to low — no fabrication.

Mock bets on MLB and college baseball

You can now place moneyline, run line (spread), and total bets on any baseball game. Settlement uses real final scores. Baseball bets flow into every analytics chart and filter on the Mock Bets dashboard.

AI Insight knows about pitchers

When you open a baseball game and ask for an AI Insight, the prompt now includes the full pitcher matchup (name, handedness, ERA/WHIP/K9, rating) and a focused instruction to treat pitching as the primary driver. Same guardrails: no betting advice, no invented stats.

Under the hood

Added a sport_registry that's now the single source of truth for team-sport metadata — the brittle "kickoff-or-tipoff" ternary is gone, and adding a sixth team sport is a single registry entry instead of a file sweep. Settlement logic for CFB/CBB/MLB/CB now dispatches through one shared helper instead of per-sport copies.

Mock Bet simulation, analytics dashboard, AI performance review

{# Mock Bets - Core #}

Mock Bet Simulation

A brand new system for tracking simulated betting decisions without real money. Place mock bets from any game detail page, track outcomes over time, and analyze your decision quality.

  • Place bets — Moneyline, spread, and total bets for CFB and CBB. Golf supports outright winner, top 5/10/20 finish, make the cut, and head-to-head matchups.
  • Auto-settlement — When a game finalizes, pending bets are automatically resolved with win/loss/push results based on actual outcomes.
  • Dashboard — My Mock Bets shows KPI cards (total bets, win rate, simulated net P/L, simulated ROI) with sport and result filters.
  • Decision review — After a bet settles, flag it as "Would Repeat" or "Would Avoid" and add reflection notes to build a feedback loop.

No real money is involved. All figures are labeled "Simulated." This is a decision-quality analysis tool.

{# Analytics Dashboard #}

Analytics Dashboard

A comprehensive analytics dashboard for your mock betting history, accessible from the "Analytics Dashboard" button on the My Mock Bets page.

  • 8 KPI cards — Total bets, W-L-P record, win rate, simulated net P/L, ROI, avg odds, avg implied probability
  • 5 interactive charts — Cumulative P/L, Rolling Win % (10-bet window), ROI by Sport, Performance by Confidence, and Odds Distribution (scatter plot)
  • Full filters — Sport, bet type, confidence level, model source, and date range
  • House vs User comparison — Head-to-head table comparing bets made with each model: win rate, ROI, avg odds, volatility
  • Confidence calibration — Expected vs actual win rate by confidence level (Low/Medium/High)
  • Edge analysis — Win rate and ROI by expected edge bucket (negative, 0-3%, 3-7%, 7%+)
{# Variance & Flat-Bet #}

Variance Testing & What-If Simulation

Stress-test your simulated results with variance metrics and a flat-bet simulation.

  • Streaks — Longest win and loss streaks
  • Drawdown — Maximum simulated drawdown from peak
  • Volatility — Standard deviation of bet-by-bet returns
  • Best/worst stretches — Best and worst N-bet stretches by P/L
  • Flat-bet simulation — Enter a flat stake and see what your results would look like with uniform sizing. Removes stake-size effects from the analysis.
{# AI Performance Commentary #}

AI Performance Commentary

Tap "Generate Commentary" on the analytics dashboard to get an AI-generated review of your simulated betting performance. The AI uses the same persona system as game insights (Analyst, New York Bookie, Southern Commentator, or Ex-Player) — set your preference in Preferences.

The AI reviews your KPIs, confidence calibration, edge analysis, and variance data to give you strengths, areas to watch, and a bottom-line assessment. Requires at least 5 settled bets.

{# Demo Data #}

Demo Mock Bets

The demo user now comes pre-loaded with ~30 mock bets (15 CFB, 10 CBB, 5 pending) so the analytics dashboard has data immediately. Settlement logs and decision reviews are included.

The separate CFB and CBB Value Boards were consolidated into a single page at /lobby/ with sport tabs. The system auto-detects which sports have upcoming games and shows only those. CBB appears first during basketball season (Nov–Apr), CFB during football season. Golf events show when available.

All authentication pages got a bold, modern redesign with a 2-column split layout. The left side features the BW logo with an entrance animation; the right side has a clean form card. On mobile it stacks vertically. Also added the full password reset flow so you can recover your account via email.

{# Analytics Pipeline #}

Analytics & Score Tracking

The analytics engine now captures model predictions automatically before each game, then records outcomes with real scores after completion. The Model Performance page shows accuracy by sport, calibration tables, time trends, and Closing Line Value (CLV).

{# Live Dashboard #}

Live Games Dashboard

The home dashboard now shows games in progress at the top with a pulsing LIVE badge and current scores. Data refreshes automatically on a 30-minute cron cycle.

{# Profile Picture #}

Profile Pictures

Upload a profile picture and it appears in the header and bottom nav. Images are center-cropped to square and compressed automatically. Works on Railway's ephemeral filesystem by storing the image directly in the database.

{# Live Data #}

Live Data Ingestion

The entire data pipeline went live. Schedules, scores, and injury reports pull from ESPN and sport-specific APIs (CFBD for football, CBBD for basketball). Betting odds come from The Odds API. Everything is optional and controlled by environment toggles — the site works perfectly with demo data when live feeds are off.

{# Security #}

Security Hardening

Public registration was disabled (invite-only access). Login is now protected against brute-force attacks with automatic lockout after 5 failed attempts. The admin panel URL was moved to a non-standard path to avoid bot scanners, and HSTS headers were enabled for production.

Enter your zip code in Preferences and all game times display in your local timezone with an abbreviation (CST, EST, etc.). Works accurately even in split-timezone states like Indiana and the Florida panhandle.

{# Season-Aware #}

Season-Aware Dashboard

The home page now only shows sports that are currently in season. During the offseason, CFB pages display a banner explaining the data is sample/demo data so there's no confusion.

Feb 7, 2026 — Launch

Where it all began

The Foundation

Brother Willies was built from scratch in a single session. The entire Django project, 7 apps, dark theme UI, and all core functionality came together at once.

What shipped on day one:

  • College Football analytics — Conferences, teams, games, odds snapshots, injury impacts, and a house model that computes win probabilities using team ratings, home-field advantage, and injuries
  • Value Board — All upcoming games ranked by the size of disagreement between the model and the market, with sorting and preference-based filtering
  • My Model — Five tunable weight sliders (rating, home field, injury, recent form, conference) that let you customize the house model and see your personalized probabilities
  • Presets — Save and load named weight configurations
  • Personal Statistics — Track your activity with summary tiles and history, filterable by time period
  • Model Performance — Accuracy and Brier score tracking from captured prediction snapshots
  • Parlay Builder — Analytical tool for evaluating multi-game parlays with combined probability, correlation detection, and a 10% vig haircut
  • Golf scaffolding — Event, golfer, and round models with a placeholder hub page
  • Context-aware help — Every page has a "?" button that opens a help panel explaining that specific page
  • Dark theme UI — Mobile-first, card-based design with responsive breakpoints and a fixed bottom navigation bar
  • Demo data — 5 conferences, 25 teams, 25 games with odds and injuries, pre-configured demo user with Alabama as favorite team

The platform was designed from day one with clear guardrails: analytics only, no betting advice, neutral language, and disclaimers on every page. These principles haven't changed.
